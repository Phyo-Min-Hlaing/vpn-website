# Kophyo.top VPN - Comprehensive Project Context

## Project Overview
**Kophyo.top VPN** is a comprehensive SaaS admin dashboard and Reseller Panel. The platform allows for the management of VPN services across multiple tenants, facilitating key generation, customer tracking, API integrations, and reseller lifecycle management.

## Tech Stack
- **Frontend Framework**: React (via React Router for SPA routing).
- **Styling**: Tailwind CSS v4.
- **UI Components**: Shadcn UI (using Radix primitives).
- **Icons**: Lucide React.
- **Expected Backend**: Node.js/Next.js APIs (currently mocking data on frontend), possibly using Supabase for persistence.

## System Architecture

### Multi-Tenant Model
The system supports multiple roles:
1. **Super Admin**: Has global access to create plans, manage API settings, and oversee all resellers.
2. **Reseller**: Has a restricted view. Can purchase keys in bulk, manage their own customers, and track their personal sales and wallet balance.

### Key Entities
- **Resellers**: Business partners who buy VPN keys at wholesale prices to sell to end-users. They have a "Wallet Balance" used for transactions.
- **Customers**: The end-users of the VPN service. Managed either directly by Admins or by specific Resellers.
- **Plans (Pricing Tiers)**: Examples include Basic 1 Month, Premium 1 Year, etc. Dictates the duration and tier of the VPN access.
- **Keys**: The actual VPN credentials generated by the system and distributed to customers.

## Key Workflows & Features

### 1. Reseller Management
- Admins can add, edit, and suspend resellers.
- Admins can view individual reseller details via the `ViewReseller` page, which includes:
  - Reseller's personal and contact info.
  - Performance metrics (Wallet, Keys Owned, Keys Sold, Profit).
  - A dedicated table of the reseller's customers, showing VPN status, usage, and expiry date.

### 2. VPN Key Management & Distribution
- Keys are generated based on selected plans.
- **Automated Messaging Integration**: Keys can be sent manually or automatically via Telegram, Viber, or SMS.
- **Renewal Reminders**: The system supports automated automated reminders via external APIs to message customers before their keys expire.

### 3. Navigation & Layout
- A persistent, collapsible sidebar layout.
- Responsive design ensuring tables and cards are usable on mobile and expansive on desktop.

## Routing Structure (React Router)
- `/` - Dashboard overview
- `/resellers` - Reseller management list
- `/resellers/add` - Form to create a new reseller
- `/resellers/edit/:resellerId` - Form to edit an existing reseller
- `/resellers/view/:resellerId` - Detailed view of a specific reseller and their customers
- `/customers` - End-user management
- `/customers/view/:customerId` - Detailed view of a specific customer
- `/keys` - VPN key management
- `/keys/create` - Generate new VPN keys
- `/plans` - Pricing tier management

## Development Guidelines for AI Context
- **Component Strategy**: Prefer using standard Shadcn UI components located in `/src/app/components/ui/`.
- **Styling Rules**: Follow the established UI aesthetic (primary color `#6B4226`, `rounded-xl`, `shadow-sm`). Never inject arbitrary inline styles when a Tailwind class is available.
- **Mock Data**: When adding new features, include robust mock arrays structured exactly like future API responses until the backend is fully wired.
- **Icons**: Always import from `lucide-react`. Ensure uniform sizing (`size={16}` for inline actions, `size={20}` for headers).
- **File Placement**:
  - Pages go in `/src/app/pages/`.
  - Reusable UI blocks go in `/src/app/components/ui/`.
  - Reusable functional components go in `/src/app/components/`.
- **Future Integration**: The backend API for SMS/Telegram/Viber messaging is the immediate next priority for implementation. The UI toggles are built, but the form submission logic needs to map to actual HTTP requests.

## Known API Connections Needed
1. **Messaging Provider**: For Telegram, Viber, SMS delivery of keys.
2. **Cron/Webhook**: For triggering renewal reminders 3-7 days before expiry.
3. **Payment Gateway**: For reseller wallet top-ups.
4. **VPN Core**: Integration with the actual VPN server infrastructure to activate/deactivate keys in real-time.
